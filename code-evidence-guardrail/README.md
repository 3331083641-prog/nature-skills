# code-evidence-guardrail

A shared overlay guardrail for keeping implementation claims faithful to code evidence.

This is not a primary writing skill. It is meant to be used together with skills such as
`nature-polishing`, `nature-paper2ppt`, or code-related documentation workflows.

The primary skill controls style, structure, audience, and output format. This guardrail controls
whether implementation claims are supported, cautious, attributed, flagged, or omitted.

Use it directly for code-faithfulness audits, implementation verification, hallucination checks
against code, or code/text consistency reviews. When used by another skill, load the full spec
only for code-grounded or verification-oriented tasks.

The full specification lives in `references/spec.md`.
